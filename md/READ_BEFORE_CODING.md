# Read Before Coding

이 폴더의 문서는 이 프로젝트에서 코드를 수정하기 전에 먼저 읽는 것을 기본 원칙으로 합니다.

## 목적

- 코드 변경 전에 현재 구조와 동작 방식을 빠르게 파악합니다.
- 기존 애니메이션 상태 전이와 입력 처리 로직을 깨뜨리지 않도록 합니다.
- 작업 전 코드리뷰 리포트에 있는 리스크와 개선 포인트를 확인합니다.

## 권장 읽기 순서

1. `docs/CODEBASE_INDEX.md`
2. `docs/CODE_REVIEW_REPORT.md`
3. 실제 구현 파일

## 작업 규칙

- `ContentView.swift`는 핵심 상태 머신, 입력 이벤트, CPU 감지, 윈도우 이동 로직이 한 파일에 집중되어 있으므로 수정 범위를 먼저 좁혀서 작업합니다.
- 애니메이션 상태를 추가하거나 수정할 때는 `PetAnimation.swift`의 `assetName`, `frameDurationsMs`, `transition`을 함께 검토합니다.
- macOS 시스템 이벤트 감지 코드는 권한, 멀티 디스플레이, 포그라운드 앱 판별 조건에 영향을 받으므로 로컬 환경 가정을 코드에 고정하지 않도록 주의합니다.
- 기능 변경 전후로 최소한 실행 흐름 기준으로 다음을 확인합니다: 앱 실행, 기본 Idle 표시, 클릭/우클릭/Force Click 반응, 작업 상태 전환, 종료 시 정리.

## 현재 문서 목록

- `docs/CODEBASE_INDEX.md`: 파일별 역할, 런타임 흐름, 핵심 의존성 정리
- `docs/CODE_REVIEW_REPORT.md`: 우선순위 기반 코드리뷰 결과와 개선 제안
